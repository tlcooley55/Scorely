Strategy
Project Summary
Scorely is a social music rating web application that allows users to search for songs, rate them on a 1Ė5 star scale, optionally write short reviews, and track their listening history. The system is not a streaming platform and does not connect to external services like Spotify. Instead, it focuses entirely on user-driven interaction and reflection.
After the third interview, the system emphasizes a simple, clean, and intuitive experience, inspired by Letterboxd. The core interaction flow is:
Search ? Rate ? Save ? Discover through friends Users can also:
? Bookmark songs for later
? View their rating history
? Follow friends and view their activity
? Showcase their Top 5 songs (manually selected)
The system avoids complexity by removing features such as messaging, notifications, algorithms, and playlists.

User Constituencies
1. Personal Trackers (Jane-type users)
? Goal: Track and organize personal music taste
? Use:
? Rating songs
? Viewing history
? Bookmarking songs
? Behavior: Less social, more reflective
2. Social Explorers (Jake-type users)
? Goal: Discover music through friends
? Use:
? Viewing friend profiles
? Exploring friend activity
? Behavior: Social, discovery-driven

Usage Contexts
? Devices: Mobile-first web app, also usable on desktop
? Environment:
? At home
? During free time
? While listening to music
? Conditions:

? Requires internet connection
? Users manually search and log songs
? No automation or background tracking

User Stories
1. Account Creation
As a new user, I want to create an account so that I can save my ratings and activity.
2. Login
As a user, I want to log into my account so that I can access my personal data.
3. Search Songs
As a user, I want to search for songs so that I can find music to rate.
4. View Song Details
As a user, I want to view a songís details so that I can decide how to rate it.
5. Rate Song
As a user, I want to rate a song from 1Ė5 stars so that I can record my opinion.
6. Write Review
As a user, I want to optionally write a short review so that I can express my thoughts.
7. Edit Rating
As a user, I want to edit my rating so that I can update my opinion over time.
8. View Rating History
As a user, I want to view my rating history so that I can track what Iíve rated.
9. Bookmark Songs
As a user, I want to bookmark songs so that I can revisit them later.
10. View Friend Profile
As a user, I want to view a friendís profile so that I can see their ratings.
11. View Friend Activity
As a user, I want to see my friendsí recent ratings so that I can discover new music.
12. Set Top 5 Songs
As a user, I want to select my top 5 songs so that I can showcase my favorites.

Structure

Data Model:


Changes:
After Interview #3, the data model stayed mostly the same, but a few important details were clarified.
The Top 5 songs feature is now confirmed to be manually selected by the user, so the TopSongs entity includes a position (1Ė5) to store the order.
The rating system was also clarified so that each user can only have one rating per song, and editing a rating replaces the old one instead of creating a new entry.
The friend-based discovery feature remains supported through the relationships between users, ratings, and songs, without requiring additional entities.
Finally, the system continues to focus only on songs, with no separate entities for albums or artists, to keep the design simple

Skeleton
Navigation Plan
Scorely uses a simple tab-based navigation system, inspired by Letterboxd, to ensure ease of use and consistency.
Primary Navigation (Persistent Tabs)
? Home
? Search
? Friends
   ? Profile Navigation Behavior
? Users can switch between tabs at any time
? Each tab represents a core section of the app
? Navigation is always visible in a top or bottom bar
? Discovery is handled through the Home and Following Feed sections, where users can see songs their friends have recently rated
Key User Flow Main Flow:
? User logs in
? Lands on Home page
? Views friend activity (discovery)
? Navigates to Search
? Searches for a song
? Clicks a song to open the Song Detail page

? Rates and optionally reviews the song on the same screen
   ? Returns to Home, Friends, or Profile Page Relationships
? Home ? Following Feed / Friend Activity ? Song Detail
? Search ? Song Detail ? Rate / Review
? Profile ? Top 5 / Rating History / Bookmarks
? Friends ? Friend Profile ? Friend Top 5 / Recent Ratings
   ? Following Feed ? Friend Ratings ? Song Detail Design Rules
? Minimize the number of pages
? Combine actions into single screens where possible
? Avoid redundant navigation
? Keep interaction simple and direct
? Make friend-based discovery easy to access from the main flow

Authentication Flow
Part 1: Narrative
All users must create an account and log in to access the system. Account Creation:
Users provide:
? Username
? Email
? Password
The system validates that the email is unique, then creates and stores the account. Login:
Users enter their email and password. The system verifies the credentials. If valid:
? The user is authenticated
   ? The user is redirected to the Home page After Login:
Once logged in, users land on the Home page where they can:
? View friend activity (discovery)
? Search for songs
   ? Navigate to their profile and friends Session Behavior:
? Users remain logged in until they log out
   ? There are no different user roles; all users have the same level of access Access Control:
Only logged-in users can:
? Rate songs

? View profiles
? Add friends
? Bookmark songs
There is no guest access to core features.

These screenshots reflect the final feedback from the client during Interview #3. Overall, most of the page structure was approved, but a few important changes were requested. The client wants the design to look more polished, with fewer separate pages related to songs. The rating and review features should be combined into the Song Detail page instead of being separate. The Top 5 songs feature was also clarified to be manually selected by the user, rather than automatically generated. In general, the final design should include album artwork, clearly visible ratings, and a clean layout similar to apps like Letterboxd. It should also support discovery through friend activity, allowing users to easily see and explore songs their friends have recently rated.
